# 升学AI助手 — 部署说明（合并版）

## 架构总览

```
┌──────────────────────────────────────┐
│  Cloudflare Worker (合并)            │
│  ┌──────────────┐  ┌──────────────┐ │
│  │ AI 代理      │  │ 用户系统     │ │
│  │ (DeepSeek)   │  │ (登录/保存)  │ │
│  ├──────────────┤  ├──────────────┤ │
│  │ 支付系统     │  │ 管理工具     │ │
│  │ (XorPay)     │  │ (生成编码)   │ │
│  └──────────────┘  └──────────────┘ │
│    ↓                                │
│  KV 命名空间: DATA                   │
│    ├── code:ACTV-XXXX → {tier,used} │
│    ├── user:ACTV-XXXX → {...}       │
│    ├── order:ORDXXXX → {...}        │
│    └── idx:xorpay:XXXX → ORDXXXX    │
└──────────────────────────────────────┘
```

所有后端功能合并为一个 Worker，一个 KV 命名空间，数据天然共享。

## 部署步骤

### 1. 创建 KV 命名空间

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 左侧菜单 → **Workers & Pages** → **KV**
3. 点击 **创建命名空间**
4. 名称填写：`DATA`
5. 创建完成，记下命名空间 ID

### 2. 创建 Worker

1. 左侧菜单 → **Workers & Pages** → **创建 Worker**
2. 名称填写：`shengxue-ai`（或任意名称）
3. 点击 **编辑代码**
4. 删除默认代码，粘贴 `workers/worker.js` 的全部内容
5. 点击 **保存并部署**

### 3. 绑定 KV 命名空间

1. Worker 详情页 → **设置** → **变量**
2. 找到 **KV 命名空间绑定** → **添加绑定**
3. **变量名称**：`DATA`（代码里使用的名字）
4. **KV 命名空间**：选择刚才创建的 `DATA`
5. 点击 **保存**

### 4. 设置环境变量

在 Worker → **设置** → **变量** → **环境变量**：

| 变量名 | 值 | 说明 |
|-------|---|------|
| `DEEPSEEK_API_KEY` | 你的 DeepSeek API Key | **点击"加密"** |
| `ADMIN_KEY` | 自行设置的密钥 | **点击"加密"**，管理后台生成激活码时使用 |
| `XORPAY_APP_ID` | XorPay 的 APP ID | — |
| `XORPAY_APP_SECRET` | XorPay 的 APP SECRET | **点击"加密"** |

添加完后，点击 **保存并部署**（会重新部署一次）。

### 5. 配置 XorPay 回调

1. 在 XorPay 后台 → **应用管理** → **支付回调地址**
2. 填写：`https://你的worker域名.workers.dev/pay/callback`
3. 保存

### 6. 配置前端页面

打开 `index.html`，找到以下代码行：

```js
var WORKER_URL = localStorage.getItem('worker_url') || 'https://你的worker域名.workers.dev';
```

将默认地址改为你的 Worker 域名。部署后用户也可以在前端设置面板中自行更改。

### 7. 部署前端

**方式一：GitHub Pages（推荐，免费）**

1. 注册 [github.com](https://github.com) → 创建仓库
2. 将项目所有文件上传到仓库
3. 仓库 → **Settings** → **Pages**
4. Source 选 **Deploy from a branch** → 选 `main` → `/ (root)`
5. 等 2 分钟即可访问

**方式二：Vercel（免费，速度快）**

1. 注册 [vercel.com](https://vercel.com) → 用 GitHub 登录
2. 导入仓库 → 部署

## 常见问题

**Q: 页面打开显示「AI服务配置中」？**
A: 检查 Worker URL 是否正确配置。点击页面右下角的 🔧 图标，填入 Worker 地址。

**Q: 支付回调不生效？**
A: 确认 XorPay 后台的回调地址填写的是 `https://你的worker域名.workers.dev/pay/callback`，且 Worker 中已配置 XORPAY_APP_ID 和 XORPAY_APP_SECRET。

**Q: 管理面板生成编码提示密钥错误？**
A: 确认 Worker 的环境变量 `ADMIN_KEY` 已设置且与前端输入的密钥一致。

**Q: 页面刷新后消息和历史丢失？**
A: 确认 Worker 地址正确且 KV 命名空间已绑定。消息会在发送后自动保存到服务端。
